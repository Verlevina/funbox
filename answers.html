Q1

Расскажите, чем, на ваш взгляд, отличается хорошая верстка от плохой с точки зрения
пользователя

Для пользователя в первую очередь важна доступность информации на сайте:

  • контент должен быть читабелен
  • блоки должны стоять на своих местах, не наезжая друг на друга
  • картинки должны быть оптимизированы
  • на фоновых изображениях задан цвет фона и цвет текта, чтобы при незагруженных картинках, текст был доступен
  • по возможности, крититческий функционал должен быть доступен при неподгруженном JS и CSS
  • Интерактивные элементы должны переключаться по TAB
  • Теги Alt и поля без подписей должны быть подписаны
  • Сайт должен загружаться быстро, код оптимизирован
  • Кроссбраузерность
  • Примененная ретинизация
менеджера проекта
  • Для менеждера в первую очередь важна верстка, при которой при изменении макета, не пришлось сильно менять код:
  • Верстка должна работать на переполнение контентом

  • Cоответствие верстки макету
  • Исправление макета не должно приводить к долгому изменению верстки

верстальщика;
  • Легкость поддержки

  • Применение популярных технологий
  • Модульность
  • Понятное именование классов
  • Отсутствие в стилях "тяжелых" селекторов
  • Использование сборщиков проектов для оптимизации работы
клиентского программиста;
  • Адекватное именование классов
  • То, что можно прописать через стили, прописано серез стили
